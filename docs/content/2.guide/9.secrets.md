vmsan provides per-project encrypted secrets using AES-256-GCM. Secrets are stored locally and injected into services at deploy time.

## Storage

Secrets are stored in two separate locations:

| Path | Contents |
| --- | --- |
| `~/.vmsan/keys/{project}.key` | Encryption key |
| `~/.vmsan/secrets/{project}.enc` | Encrypted secret data |

## Set secrets

```bash
vmsan secrets set DB_PASSWORD=mysecret API_KEY=abc123
```

You can set multiple secrets in a single command.

## List secrets

```bash
vmsan secrets list
```

::note
Only secret names are displayed. Values are never shown in list output.
::

## Remove secrets

```bash
vmsan secrets unset DB_PASSWORD API_KEY
```

You can remove multiple secrets in a single command.

## Using secrets in vmsan.toml

Reference secrets with the `${{ secrets.KEY }}` syntax:

```toml
[services.api]
runtime = "node24"
env = { DATABASE_URL = "${{ secrets.DATABASE_URL }}" }

[accessories.db]
type = "postgres"
env = { POSTGRES_PASSWORD = "${{ secrets.DB_PASSWORD }}" }
```

Secrets are resolved and injected at deploy time. The `${{ secrets.KEY }}` references are never sent to VMs as literal strings.

## Flags

| Flag | Default | Description |
| --- | --- | --- |
| `--config` | `vmsan.toml` | Path to vmsan.toml |
| `--project` | from vmsan.toml | Project name override |

## Security

- **AES-256-GCM encryption** with a random IV generated per write
- **Key separation** — encryption keys are stored separately from encrypted data
- **No value display** — secret values are never shown in `secrets list` output
- **Deploy-time injection** — secrets are injected when services start, not stored in vmsan.toml

::warning
The encryption keys in `~/.vmsan/keys/` must be kept secure. Anyone with access to both the key file and the encrypted data file can decrypt your secrets.
::

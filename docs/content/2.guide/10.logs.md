vmsan provides log streaming from running VMs, including application output and DNS query logs.

## Stream logs

View recent logs from a VM:

```bash
vmsan logs <vmId>
```

## Follow log output

Stream logs in real time, similar to `tail -f`:

```bash
vmsan logs -f <vmId>
```

## Control output

Show a specific number of historical lines:

```bash
vmsan logs -n 50 <vmId>
```

Show timestamps alongside log entries:

```bash
vmsan logs -t <vmId>
```

## Flags

| Flag | Short | Default | Description |
| --- | --- | --- | --- |
| `--lines` | `-n` | `100` | Number of historical lines to show |
| `--follow` | `-f` | `false` | Follow log output in real time |
| `--timestamps` | `-t` | `false` | Show timestamps with each line |

## DNS logs

Inspect DNS queries made by a VM's dnsproxy sidecar:

```bash
vmsan logs --dns <vmId>
```

This shows the query name, response code, and resolved IPs for each DNS request. DNS logs are useful for debugging network policy issues, especially when using `custom` network policies with allowed domains.

::tip
If a service cannot reach an external API, check DNS logs first. In `custom` network policy mode, queries for non-allowed domains return `NXDOMAIN`, which often appears as a connection failure in application logs.
::

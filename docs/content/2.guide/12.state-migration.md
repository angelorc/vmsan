vmsan now uses SQLite (WAL mode) for state storage instead of JSON files. The `vmsan migrate` command imports existing JSON state into the new database.

## Run the migration

Preview what would be migrated before making changes:

```bash
vmsan migrate --dry-run
```

When ready, run the migration:

```bash
vmsan migrate
```

VM state files from `~/.vmsan/` are imported into the SQLite database at `~/.vmsan/state.db`.

## SQLite tables

The database contains the following tables:

| Table | Description |
| --- | --- |
| `vms` | VM metadata and state |
| `hosts` | Registered hosts (multi-host) |
| `projects` | Project configuration |
| `mesh_allocations` | Mesh IP assignments |
| `sync_log` | Sync engine command log |

::note
New installations use SQLite by default. Migration is only needed when upgrading from pre-0.4.0 installations that used JSON state files.
::

::warning
Back up your `~/.vmsan/` directory before running `vmsan migrate`. While the migration does not delete original JSON files, having a backup ensures you can recover if something goes wrong.
::

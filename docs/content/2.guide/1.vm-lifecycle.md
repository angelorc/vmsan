vmsan provides commands to manage the full lifecycle of Firecracker microVMs.

## Create a new VM

Use `vmsan create` to spin up a new microVM:

```bash
vmsan create --runtime node22 --memory 512 --vcpus 2
```

You can configure the VM with these flags:

| Flag           | Default | Description                                       |
| -------------- | ------- | ------------------------------------------------- |
| `--vcpus`      | `1`     | Number of vCPUs                                   |
| `--memory`     | `128`   | Memory in MiB                                     |
| `--runtime`    | `base`  | Runtime label (`base`, `node22`, `node24`, `python3.13`) |
| `--disk`       | `10gb`  | Root disk size                                    |
| `--timeout`    | none    | Auto-shutdown timeout (e.g. `1h`, `30m`)          |
| `--connect`    | `false` | Automatically connect to the shell after creation |
| `--project`    | none    | Project label for grouping VMs                    |
| `--snapshot`   | none    | Snapshot ID to restore from                       |
| `--connect-to` | none    | Mesh connections (comma-separated `service:port`) |
| `--service`    | none    | Register this VM as a mesh service                |
| `--host`       | none    | Remote host for multi-host deployment             |
| `--allow-icmp` | `false` | Allow ICMP traffic (ping)                         |

::tip
Add `--connect` to immediately open a shell session after the VM is created.
::

## Create a VM from a Docker image

Build a root filesystem from any Docker or OCI image:

```bash
vmsan create --from-image node:22-alpine
```

See the [Docker Images](/guide/docker-images) guide for more details.

## List all VMs

```bash
vmsan list
```

Use the alias `vmsan ls` for a shorter command. Add `--json` for structured output.

## Start a stopped VM

```bash
vmsan start <vm-id>
```

## Stop a running VM

```bash
vmsan stop <vm-id> [vm-id...]
```

You can stop multiple VMs at once by passing additional IDs.

## Remove a VM

```bash
vmsan remove <vm-id> [vm-id...]
```

Use the alias `vmsan rm` for a shorter command. You can remove multiple VMs at once by passing additional IDs. Use `--force`/`-f` to skip the confirmation prompt.

::caution
Removing a VM deletes its chroot directory and state file permanently.
::

## Execute commands

Run a command inside a VM and stream the output:

```bash
vmsan exec <vm-id> ls -la /tmp
```

Use `-i` for interactive mode (full PTY), `--sudo` to run as root, `-w` to set the working directory, and `-e` to inject environment variables:

```bash
# Interactive session
vmsan exec -i <vm-id> bash

# Run as root
vmsan exec --sudo <vm-id> apt-get update

# With working directory and env
vmsan exec -w /app -e NODE_ENV=production <vm-id> node server.js
```

## Connect to a running VM

Open an interactive shell session:

```bash
vmsan connect <vm-id>
```

This opens a WebSocket PTY connection to the in-VM agent, providing a full terminal experience. If the connection drops, the client automatically retries up to 10 times with exponential backoff.

## Logs

View logs for a VM or its services:

```bash
vmsan logs <vm-id> [service]
```

See the [Logs](/guide/logs) guide for details on filtering, following, and timestamps.

::note
The vmsan-gateway daemon manages VM lifecycle. It starts automatically when needed.
::

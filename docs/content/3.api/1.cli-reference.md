## Global Flags

| Flag        | Description                |
| ----------- | -------------------------- |
| `--json`    | Output structured JSON     |
| `--verbose` | Show detailed debug output |
| `--version` | Show version information   |

## VM Management

### `vmsan create`

Create and start a Firecracker microVM.

| Flag               | Type   | Default     | Description                                          |
| ------------------ | ------ | ----------- | ---------------------------------------------------- |
| `--vcpus`          | int    | `1`         | Number of vCPUs                                      |
| `--memory`         | int    | `128`       | Memory in MiB                                        |
| `--runtime`        | string | `base`      | Runtime (`base`, `node22`, `node24`, `python3.13`)   |
| `--disk`           | string | `10gb`      | Root disk size                                       |
| `--kernel`         | string | —           | Path to kernel image (auto-detected)                 |
| `--rootfs`         | string | —           | Path to rootfs image (auto-detected)                 |
| `--timeout`        | string | —           | Auto-shutdown timeout (`30s`, `5m`, `1h`, `2h30m`)   |
| `--project`        | string | —           | Project label for grouping VMs                       |
| `--snapshot`       | string | —           | Snapshot ID to restore from                          |
| `--from-image`     | string | —           | Build rootfs from Docker/OCI image                   |
| `--publish-port`   | string | —           | Ports to forward (comma-separated)                   |
| `--network-policy` | string | `allow-all` | Network mode: `allow-all`, `deny-all`, `custom`      |
| `--allowed-domain` | string | —           | Allowed domains (comma-separated, wildcard `*`)      |
| `--allowed-cidr`   | string | —           | Allowed CIDR ranges (comma-separated)                |
| `--denied-cidr`    | string | —           | Denied CIDR ranges (comma-separated)                 |
| `--bandwidth`      | string | —           | Max bandwidth (e.g. `50mbit`)                        |
| `--allow-icmp`     | bool   | `false`     | Allow ICMP traffic (ping)                            |
| `--no-seccomp`     | bool   | `false`     | Disable seccomp-bpf filter                           |
| `--no-pid-ns`      | bool   | `false`     | Disable PID namespace isolation                      |
| `--no-cgroup`      | bool   | `false`     | Disable cgroup resource limits                       |
| `--connect-to`     | string | —           | Mesh connections (service:port pairs, comma-separated)|
| `--service`        | string | —           | Register as mesh service name                        |
| `--host`           | string | —           | Remote host server address (multi-host)              |
| `--connect`        | bool   | `false`     | Open shell after creation                            |

::note
`--from-image` and `--connect` are mutually exclusive. Custom images do not include the agent.
::

### `vmsan list`

List all VMs. Alias: `ls`.

### `vmsan start`

Start a stopped VM.

**Usage:** `vmsan start <vmId>`

### `vmsan stop`

Stop one or more running VMs.

**Usage:** `vmsan stop <vmId> [vmId...]`

### `vmsan remove`

Remove one or more VMs. Alias: `rm`.

**Usage:** `vmsan remove <vmId> [vmId...]`

| Flag         | Type | Default | Description                          |
| ------------ | ---- | ------- | ------------------------------------ |
| `--force, -f`| bool | `false` | Force removal (stops running VMs first) |

## VM Interaction

### `vmsan connect`

Open interactive shell to a running VM.

**Usage:** `vmsan connect <vmId>`

| Flag              | Type   | Default | Description                        |
| ----------------- | ------ | ------- | ---------------------------------- |
| `--session`, `-s` | string | —       | Attach to existing shell session ID |

::tip
Auto-reconnects up to 10 times with exponential backoff on WebSocket drops.
::

### `vmsan exec`

Execute a command inside a running VM.

**Usage:** `vmsan exec <vmId> <command> [args...]`

| Flag                    | Type     | Default | Description                           |
| ----------------------- | -------- | ------- | ------------------------------------- |
| `--sudo`                | bool     | `false` | Run as root                           |
| `-i`, `--interactive`   | bool     | `false` | Interactive shell mode (PTY)          |
| `-t`, `--tty`           | bool     | `false` | Allocate pseudo-TTY (compatibility)   |
| `-w`, `--workdir`       | string   | —       | Working directory inside VM           |
| `-e`, `--env`           | string[] | —       | Environment variable `KEY=VAL` (repeatable) |
| `--no-extend-timeout`   | bool     | `false` | Skip timeout extension                |

### `vmsan upload`

Upload local files to a running VM.

**Usage:** `vmsan upload <vmId> <files...>`

| Flag           | Type   | Default | Description                      |
| -------------- | ------ | ------- | -------------------------------- |
| `-d`, `--dest` | string | `/root` | Destination directory inside VM  |

### `vmsan download`

Download a file from a running VM.

**Usage:** `vmsan download <vmId> <remotePath>`

| Flag           | Type   | Default          | Description          |
| -------------- | ------ | ---------------- | -------------------- |
| `-d`, `--dest` | string | cwd/basename     | Local destination path |

### `vmsan logs`

Stream logs from a running VM.

**Usage:** `vmsan logs <vmId> [service]`

| Flag              | Type | Default | Description              |
| ----------------- | ---- | ------- | ------------------------ |
| `-n`, `--lines`   | int  | `100`   | Number of historical lines |
| `-f`, `--follow`  | bool | `false` | Follow log output        |
| `-t`, `--timestamps` | bool | `false` | Show timestamps       |
| `--dns`           | bool | `false` | Show DNS query logs      |

## Configuration & Deployment

### `vmsan init`

Initialize a `vmsan.toml` configuration file with auto-detection.

**Usage:** `vmsan init`

| Flag    | Type | Default | Description                            |
| ------- | ---- | ------- | -------------------------------------- |
| `--yes` | bool | `false` | Non-interactive mode with defaults     |

### `vmsan validate`

Validate a `vmsan.toml` configuration file.

**Usage:** `vmsan validate [path]`

Default path: `vmsan.toml`

### `vmsan up`

Deploy all services defined in `vmsan.toml`.

**Usage:** `vmsan up`

| Flag       | Type   | Default      | Description          |
| ---------- | ------ | ------------ | -------------------- |
| `--config` | string | `vmsan.toml` | Path to `vmsan.toml` |

### `vmsan deploy`

Re-deploy services (upload, build, start) without recreating VMs.

**Usage:** `vmsan deploy [service]`

| Flag       | Type   | Default      | Description          |
| ---------- | ------ | ------------ | -------------------- |
| `--config` | string | `vmsan.toml` | Path to `vmsan.toml` |

### `vmsan down`

Stop all project services.

**Usage:** `vmsan down`

| Flag        | Type   | Default      | Description                  |
| ----------- | ------ | ------------ | ---------------------------- |
| `--config`  | string | `vmsan.toml` | Path to `vmsan.toml`         |
| `--destroy` | bool   | `false`      | Also remove VMs and data     |
| `--force`   | bool   | `false`      | Skip confirmation prompt     |

### `vmsan status`

Show project service status overview.

**Usage:** `vmsan status`

| Flag       | Type   | Default      | Description          |
| ---------- | ------ | ------------ | -------------------- |
| `--config` | string | `vmsan.toml` | Path to `vmsan.toml` |

## Network

### `vmsan network update`

Update network policy for a running VM.

**Usage:** `vmsan network update <vmId>`

| Flag           | Type   | Description                                     |
| -------------- | ------ | ----------------------------------------------- |
| `--policy`     | string | Network policy (`allow-all`, `deny-all`, `custom`) |
| `--domains`    | string | Comma-separated allowed domains                 |
| `--cidrs`      | string | Comma-separated allowed CIDRs                   |
| `--allow-icmp` | bool   | Allow ICMP traffic                               |

### `vmsan network connections`

Show mesh network connections for current project.

**Usage:** `vmsan network connections`

| Flag       | Type   | Default      | Description          |
| ---------- | ------ | ------------ | -------------------- |
| `--config` | string | `vmsan.toml` | Path to `vmsan.toml` |

## Snapshots

### `vmsan snapshot create`

Create a snapshot of a running VM.

**Usage:** `vmsan snapshot create <vmId>`

### `vmsan snapshot list`

List snapshots. Alias: `ls`.

**Usage:** `vmsan snapshot list [vmId]`

### `vmsan snapshot delete`

Delete a snapshot. Alias: `rm`.

**Usage:** `vmsan snapshot delete <snapshotId>`

**Usage:** `vmsan secrets set <KEY=VALUE...>`

| Flag        | Type   | Default      | Description            |
| ----------- | ------ | ------------ | ---------------------- |
| `--config`  | string | `vmsan.toml` | Path to `vmsan.toml`   |
| `--project` | string | —            | Project name override  |

### `vmsan secrets list`

List secret names (values not shown). Alias: `ls`.

**Usage:** `vmsan secrets list`

| Flag        | Type   | Default      | Description            |
| ----------- | ------ | ------------ | ---------------------- |
| `--config`  | string | `vmsan.toml` | Path to `vmsan.toml`   |
| `--project` | string | —            | Project name override  |

### `vmsan secrets unset`

Remove one or more secrets.

**Usage:** `vmsan secrets unset <KEY...>`

| Flag        | Type   | Default      | Description            |
| ----------- | ------ | ------------ | ---------------------- |
| `--config`  | string | `vmsan.toml` | Path to `vmsan.toml`   |
| `--project` | string | —            | Project name override  |

## Multi-Host

### `vmsan hosts add`

Register a new host with the control plane.

**Usage:** `vmsan hosts add`

| Flag       | Type   | Required | Description                    |
| ---------- | ------ | -------- | ------------------------------ |
| `--server` | string | yes      | Control plane server address   |
| `--token`  | string | yes      | Authentication token           |
| `--name`   | string | no       | Host display name              |

### `vmsan hosts list`

List registered hosts. Alias: `ls`.

### `vmsan hosts remove`

Remove a host from the cluster. Alias: `rm`.

**Usage:** `vmsan hosts remove <hostId>`

| Flag       | Type   | Required | Description                    |
| ---------- | ------ | -------- | ------------------------------ |
| `--server` | string | yes      | Control plane server address   |
| `--token`  | string | no       | Authentication token           |

### `vmsan hosts check`

Health check hosts.

**Usage:** `vmsan hosts check [hostName]`

## System

### `vmsan doctor`

Check system prerequisites and installation health. Returns exit code `1` if any check fails.

::tip
Use `vmsan doctor --json` for scripted health checks in CI pipelines.
::

### `vmsan migrate`

Migrate JSON VM state files to SQLite database.

**Usage:** `vmsan migrate`

| Flag        | Type | Default | Description                      |
| ----------- | ---- | ------- | -------------------------------- |
| `--dry-run` | bool | `false` | Preview without making changes   |

### `vmsan gateway`

Start the vmsan gateway daemon (privileged).

### `vmsan server`

Start the vmsan control plane server.

### `vmsan agent-host`

Start the agent-host worker for multi-host clusters.

## SDK

vmsan exports a TypeScript SDK for programmatic use:

```ts
import { AgentClient, SecretsStore } from "vmsan";
```

Key exports: `AgentClient`, `SecretsStore`, `SqliteVmStateStore`, `SnapshotService`, `CloudflareService`, `createVmsan`, `createVmsanContext`.

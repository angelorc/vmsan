## Prerequisites

- Linux (x86_64 or aarch64) with KVM support
- Docker (only required for `--from-image` and local runtime rebuilds)

::note
vmsan no longer requires npm, bun, or any JavaScript runtime. The CLI and all components are standalone Go binaries.
::

## Install vmsan

Run the install script to download everything you need:

```bash
curl -fsSL https://vmsan.dev/install | bash
```

This single command installs:

- **Firecracker + Jailer** to `/usr/local/bin` (latest release)
- **vmsan CLI** to `/usr/local/bin`
- **vmsan-gateway** to `/usr/local/bin`
- **Linux kernel** (vmlinux 6.1) to `~/.vmsan/`
- **Ubuntu 24.04 rootfs** (converted from squashfs to ext4) to `~/.vmsan/`
- **Runtime images** (`node22`, `node24`, `python3.13`) downloaded from `artifacts.vmsan.dev`
- **seccomp filter** (`seccompiler-bin` from the Firecracker tarball)
- **vmsan group** for Unix socket permissions

::tip
The install script creates a `vmsan` system group and configures socket permissions so the CLI can communicate with the gateway without requiring sudo.
::

## Gateway setup

The vmsan-gateway runs as a systemd service and starts automatically when needed. It manages VM lifecycle, networking, and firewalls as a privileged daemon.

The gateway listens on a Unix socket at `/run/vmsan/gateway.sock`. The CLI communicates with it over gRPC --- you never need to run `vmsan` with sudo.

::note
If you need to manually manage the gateway service:

```bash
sudo systemctl status vmsan-gateway
sudo systemctl restart vmsan-gateway
sudo journalctl -u vmsan-gateway -f
```
::

## Build the in-VM agent

The Go agent runs inside each VM and handles shell access, file operations, and command execution.

Release installs download `vmsan-agent` automatically. For source installs:

```bash
cd agent
make install
cd ..
```

## Verify the installation

```bash
vmsan doctor
```

This checks that all components are installed correctly: Firecracker, the gateway, KVM access, kernel, rootfs, and runtime images.

::tip
For staging or recovery testing, you can override the runtime manifest:

```bash
curl -fsSL https://vmsan.dev/install | \
  VMSAN_RUNTIME_MANIFEST_URL=https://artifacts.vmsan.dev/runtimes/channels/staging.json bash
```

To force local runtime builds during a release install:

```bash
curl -fsSL https://vmsan.dev/install | \
  VMSAN_FORCE_LOCAL_RUNTIME_BUILD=1 bash
```

To install a specific branch for testing:

```bash
curl -fsSL https://vmsan.dev/install | bash -s -- --ref feat/my-branch
```
::

## Uninstall

To remove all vmsan components:

```bash
curl -fsSL https://vmsan.dev/install | bash -s -- --uninstall
```

This removes Firecracker, the vmsan binaries, kernel, rootfs, and runtime images from `~/.vmsan/` and `/usr/local/bin`.
